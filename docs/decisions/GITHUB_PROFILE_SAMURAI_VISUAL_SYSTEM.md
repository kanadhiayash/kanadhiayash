**Status:** APPROVED FOR IMPLEMENTATION REVIEW  
**Approved by:** Yash Kanadhia  
**Decision date:** 2026-07-11  
**Applies to:** GitHub profile hero and supporting profile identity assets

## Objective

Give the profile a minimal samurai-inspired identity without changing the approved public facts, portfolio information architecture, evidence structure, or project claims.

## Locked hero facts

The hero must preserve these values exactly:

- Yash Kanadhia
- Product Designer
- Toronto, Ontario, Canada
- I design and build systems that connect people to outcomes.

No visual redesign may alter, replace, embellish, or contradict these facts.

## Locked metrics

Use these exact metrics:

- 2 Ready to Ship Projects
- 1 Team Project
- 5 Certificates

The previous wording `2 shipped projects` is retired from the hero.

## Locked code panels

The hero uses concise code-style panels for Zeref Memory Engine and PerFin OS.

Zeref may state:

- local-first
- guarded writes
- cross-harness handoffs

PerFin may state:

- team ownership
- guest and authenticated workspaces
- React Native and Firebase platform

The panels must not imply production deployment, solo ownership of PerFin OS, or capabilities that are not verified.

## Locked tool rail

Display the tools in this order:

1. Figma
2. Linear
3. Notion
4. React
5. SwiftUI
6. Firebase
7. Claude
8. Codex
9. GitHub

Operational meaning:

- Figma: product design, prototyping, systems, and handoff
- Linear: primary project and execution tracker
- Notion: miscellaneous notes, references, and supporting planning
- React: web product implementation
- SwiftUI: native Apple product implementation
- Firebase: authentication, data, storage, and application services
- Claude: product thinking, design exploration, research, and development support
- Codex: implementation, repository work, testing, and technical review
- GitHub: source control, pull requests, CI, evidence, and public proof

## Visual direction

The visual system is minimal samurai editorial, not cosplay, anime, or game interface styling.

Required characteristics:

- lacquer-black and rice-paper modes
- muted crimson accent
- restrained enso, torii, grid, and blossom references
- strong negative space
- slow ambient motion only
- identical information architecture in light and dark modes
- readable without animation
- no fake interactive buttons inside the hero
- no decorative Japanese text used without verified meaning

## Theme behavior

The motion SVG must adapt to the viewer's light or dark preference while preserving the same layout and content.

The static fallback uses the approved dark composition and exists for accessibility and graceful degradation.
